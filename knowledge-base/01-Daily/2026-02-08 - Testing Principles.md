## What I Learned
The syllabus mentions several more proposed testing principles, from this syllabus only gives seven of them.
1. Testing shows the presence, not the absence of defects (Buxton 1970)
2. Exhaustive testing is impossible (Manna 1978)
3. Early testing saves time and money in SDLC (Boehm 1981)
4. Defects cluster together (Enders 1975) (Pareto principle)
5. Tests wear out (Beizer 1990) - useful in atomated regression testing
6. Testing is context dependent (Kaner 2011)
7. Absence-of-defects fallacy (Boehm 1981) - also need verification and validation

Pareto principle: 
- 80/20 rule
- write smart tests (20%) the finds 80% of bugs
- exploratory testing on high-risk area
- automate critical 20% tests first
- smoke test for core functionality
- identify risk hotspots
- prioritize functional over cosmetic
