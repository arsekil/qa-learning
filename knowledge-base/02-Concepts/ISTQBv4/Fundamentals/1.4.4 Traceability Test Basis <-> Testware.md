**K2**

Test monitoring and test control => effective implementation => establish and maintain **traceability** throughout test process => between => test basis elements, testware(e.g. test conditions, risks, test cases) => test results => defects

**traceability** supports coverage evaluation => coverage criteria can be used as **KPI**, to show to what extent test objectives have been achieved
e.g.: 
1. Traceability of test cases to requirements can verify that the requirements are covered by test cases
2. Traceability of test results to risks can be used to evaluate the level of residual risk in a test object

evaluating coverage + **traceability** => determine the impact of changes, facilitates audits and helps meet IT governance criteria
**traceability** => test progress reports and test completion reports easy to understand => including status of test basis elements
**traceability** assists in communicating technical aspects to stakeholders in understandable manner
**traceability** provides information to assess product quality, process capability and project progress against business goals
